= Validation API  

The Validation API is part of the TED Central Validation Service (TED CVS), which ensures that procurement notices meet compliance standards. It allows users to check notices for errors and warnings independently at any time, ensuring they are correct before submission for publication.


== Available Operations

//[.doc h3]
//* Validate Procurement Notices

=== Validate Procurement Notices

The Validation API ensures procurement notices meet EU standards by identifying errors and issuing warnings for potential legal issues. This independent service can validate notices anytime before theyâ€™re sent for publication, ensuring they are technically ready for TED. Any notice that passes validation through CVS upon submission to the Publication API or the eNotices2 front-end, can be considered technically valid for publication on TED.


==== API Request
[source]
----
POST /v3/notices/validate
----

== Who Uses This API

* *Developers*: Ensuring notices meet requirements, especially before submission. 
* *eSenders*: Avoiding compliance errors during submission. 
* *eNotices2 application*: For validating notices before or upon submission.
