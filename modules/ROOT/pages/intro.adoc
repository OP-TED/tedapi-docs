= TED API

APIs, or Application Programming Interfaces, are tools that allow different software applications to communicate with each other. TED API provides a set of programmatic tools to simplify and improve procurement processes. These APIs work with the TED applications (TED Apps) to make procurement more efficient. From version 3, TED API brings all services together under a unified gateway for seamless use.

== Available APIs and Their Purposes

TED APIs are designed to meet specific needs within the TED procurement ecosystem. Below is an overview of each API and its purpose:

=== Conversion API

* Enables the transition from legacy TED XML notices to modern eForms for users and systems alike.

xref:convert.adoc[Read more about the Conversion API]

=== Developer Operations API

* Offers tools to help developers integrate smoothly with TED APIs, including programmatic API key renewal.

=== Publication API

* Automates the management and submission of procurement notices.
* Works seamlessly with the eNotices2 system for efficient handling.

xref:publish.adoc[Read more about the Publication API]

=== Search API

* Provides access to published procurement notices.
* Enables data retrieval and analysis through expert queries and bulk downloads.

xref:search.adoc[Read more about the Search API]

=== Validation API

* Checks procurement notices for compliance with EU standards.
* Identifies errors and warnings before and upon submission or independently.

xref:validate.adoc[Read more about the Validation API]

=== Visualisation API

* Transforms procurement notices into user-friendly formats like PDFs or HTML.
* Supports both public dissemination and internal use.

xref:visualise.adoc[Read more about the Visualisation API]

'''

These APIs support diverse procurement needs, ensuring efficiency, compliance, and accessibility. For detailed guidance, please see our full documentation.
